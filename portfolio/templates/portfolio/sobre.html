Sobre esta Aplicação

1. Arquitetura MVT do Django

O Django segue o padrão MVT (Model-View-Template), uma variante do padrão MVC:

  • Model — define a estrutura dos dados e comunica com a base de dados
  • View — recebe o pedido HTTP, consulta os modelos e retorna uma resposta
  • Template — ficheiro HTML que apresenta os dados ao utilizador

Fluxo: Browser → URLs → View → Model → Base de Dados → View → Template → Browser

2. Modelação de Dados

A aplicação tem as seguintes entidades principais:

  • Licenciatura — curso universitário
  • UnidadeCurricular — disciplina pertencente a uma licenciatura
  • Docente — professor que leciona UCs
  • Projeto — projeto desenvolvido numa UC
  • Tecnologia — tecnologia usada nos projetos
  • TFC — trabalhos finais de curso
  • Competencia — competências adquiridas
  • Formacao — percurso formativo
  • MakingOf — registo do processo de desenvolvimento

models.py

class Tecnologia(models.Model):
    nome = models.CharField(max_length=50)
    tipo = models.ForeignKey(TipoTecnologia, ...)
    descricao = models.TextField()
    opiniao = models.TextField()
    nivel_interesse = models.IntegerField()

class Projeto(models.Model):
    nome = models.CharField(max_length=100)
    tecnologias = models.ManyToManyField(Tecnologia)
    unidade_curricular = models.ForeignKey(UnidadeCurricular, ...)
