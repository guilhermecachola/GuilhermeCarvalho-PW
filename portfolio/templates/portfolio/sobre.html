1. Arquitetura MVT do Django

O Django utiliza o padrão MVT (Model-View-Template) para organizar o código:

  • Model: Define a estrutura dos dados (Base de Dados).
  • View: Gere a lógica e processa os pedidos do utilizador.
  • Template: Define a interface final apresentada ao utilizador (HTML).

2. Modelação de Dados

A aplicação foi modelada para representar a estrutura académica e profissional, interligando unidades curriculares, projetos e competências.

Implementação (models.py)

Foram utilizados campos como ManyToManyField para associar projetos a múltiplas tecnologias e ForeignKey para a hierarquia das UCs.
